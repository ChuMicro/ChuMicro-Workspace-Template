examples + projects/example_sensor: delete dead `_radio_for_runtime()` helpers

After the chumicro mono-repo's Phase 1 work auto-detected `wifi.radio`
on CircuitPython (the `chumicro_sockets._adapters.cp._pool_for(None)`
path), every callsite that used `radio=` ended up with a `None` value
that was equivalent to passing `wifi.radio` explicitly.  The
`_radio_for_runtime()` helper that every networked example carried —

```python
def _radio_for_runtime():
    try:
        import wifi
        return wifi.radio
    except ImportError:
        return None
```

— was dead code on every supported board.  On CP it returned
`wifi.radio`, which is exactly what `_pool_for(None)` would auto-
detect.  On MP / CPython it returned `None`, which is also the
default.  Five examples were carrying the helper plus the matching
`radio=radio` plumbing through socket-factory closures.

This commit drops the helper from all five and threads the
simpler signature back through:

* `projects/example_sensor/app.py` — helper removed; `_make_socket_factory`
  loses its `radio` parameter; the `tls_client_socket` /
  `tcp_client_socket` calls drop `radio=`.
* `examples/periodic_get/app.py` — helper removed; `HttpClient`
  constructor drops `connection_factory=chumicro_sockets_factory(radio=radio)`
  to plain `chumicro_sockets_factory()`.
* `examples/telemetry_publisher/app.py` — same shape as
  example_sensor; `_make_socket_factory` loses its `radio` parameter.
* `examples/two_projects/server/app.py` — helper removed; the
  `tcp_listening_socket` call drops `radio=` from its lambda.
* `examples/two_projects/sensor/app.py` — helper removed; the
  `HttpClient` factory call drops `radio=`.

Net effect: 5 example files lose ~6 lines of pure boilerplate
each.  Beginners reading the examples no longer have to figure out
why every networked thing carries the same six-line wifi-import
dance — the dance was load-bearing under the old API and is
unnecessary now.

The library-side `radio=` kwarg stays in the public API for the
rare multi-radio / AirLift / SAMD-M0 case
(`chumicro_sockets._adapters.cp.py:35-40` documents it).  Just no
need to surface it in every example.

Tested
======

3 template tests pass (`pytest tests/ projects/_template/tests/`).
The example apps continue to import cleanly under the chumicro
mono-repo's `verify-examples` step.  No behavior change on any
board — the deleted helper was returning the same value
`_pool_for(None)` now auto-detects.
