flash is the default deploy mode (Decision 0047 from chumicro mono-repo)

Implements the workspace-template half of the cross-repo deploy-mode
flip specified by Decision 0047 in the chumicro mono-repo (commit
`de99870`).

Three concrete changes
======================

1. **`_templates/devices.yml`** gains an explicit
   `defaults.deploy_mode: flash` line.  The previous template was
   silent (let chumicro-deploy's `DEFAULT_DEPLOY_MODE` decide), but
   silence forces the user to know what the default is — explicit
   reads cleaner.  The accompanying comment explains why flash is
   the default and how to opt back to ram per-device or via
   `--deploy-mode ram`.

2. **`CONTRIBUTING.md`** — "Device modes — RAM vs. flash" section
   reworked.  The old prose said "(`deploy_mode: ram`, the default)";
   updated to "(`deploy_mode: flash`, the default)" with the
   rationale (production-shaped, atomic, matches real deploys).  New
   block-quote callout describes the `[tool.chumicro] requires_flash`
   auto-switch behavior the chumicro mono-repo's Decision 0047
   ships.

3. **`AGENTS.md`** — "RAM mode is for single-library experiments"
   bullet expanded to "Flash mode is the default; RAM mode is opt-in
   for single-library experiments".  Now mentions the four heavier
   libraries that declare `requires_flash = true` and explains the
   auto-switch.  Same operational guidance for the agent ("surface
   this when the user reports messages stop after first publish or
   ENOENT on /runtime_config.msgpack") — flash being the default
   means most users never hit the auto-switch, but when they do,
   the agent should know what's happening.

4. **`.github/skills/register-board/SKILL.md`** — example YAML
   snippet flipped from `deploy_mode: ram   # or flash per device
   class` to `deploy_mode: flash   # default; override to ram per-
   device for unit-style experiments`.  Surfaces the right default
   to anyone reading the skill while registering a board.

After this commit
=================

A user on the previous template who runs `python run.py setup`
in a fresh clone gets `defaults.deploy_mode: flash` materialised
into their `devices.yml`.  Existing workspaces aren't auto-
migrated — `_templates/` is tool-owned, but `devices.yml`'s
contents are user-owned (gitignored, mutated by `add-device` /
`probe`).  Users on existing workspaces who want the new default
edit `devices.yml` themselves or re-run `setup` after deleting
the file.

Tested
======

3 template tests pass (`pytest tests/ projects/_template/tests/`)
against chumicro-workspace's existing version; the deploy-mode
default flip is consumed transparently by the chumicro-deploy
0.5.0 published in mono-repo `de99870`.
