rename `libs/` -> `shared/` (with README), drop empty `libraries/`, packages/ now ships with README

Phase 2 / item 2 of the chumicro mono-repo's beginner-comfort onramp.
Implements the template-repo half of the cross-repo workspace-shape
update specified by Decision 0046 in the chumicro mono-repo
(commit `283918b`).

Three concrete changes
======================

1. **`libs/` -> `shared/`** — rename via `git mv libs shared`.  The
   directory is empty in the default template (placeholder for
   user code), so the move is mostly cosmetic + a few prose
   touch-ups.  The new name reads as "shared between my projects"
   rather than "library helpers" (which conflated with `libraries/`).
   Replace the empty-folder `.gitkeep` with `shared/README.md`
   that explains what goes in here, when to use it vs `libraries/`,
   and the implicit-namespace-package import shape.

2. **`libraries/` removed from the default template** (it was never
   committed, so no actual file change).  The folder materialises
   lazily — `python run.py new --library <name>` creates it the
   first time the user opts in.  Most workspaces will never have a
   `libraries/` and shouldn't have to learn what it is up-front.

3. **`packages/` now ships with a `README.md`** — same pattern as
   `shared/`.  The folder already shipped with a self-ignoring
   `.gitignore`; this commit extends the gitignore to except the
   README so the README is committed even though the rest of the
   folder's contents are not.  Honest framing: `packages/` is a
   manual-drop area for third-party Python source trees the
   workspace's projects import on the device — typically
   MicroPython-only or CircuitPython-only libraries that don't
   ship to PyPI in any installable form, or vendored copies of
   upstream forks.  Not a package manager and not a contrast to
   pip; just a place to drop source trees the import-graph walker
   should know about.

Files touched
=============

* `libs/` -> `shared/` (git rename).
* `shared/.gitkeep` removed; `shared/README.md` added.
* `packages/README.md` added; `packages/.gitignore` extended to
  except both itself and the README.
* `README.md` — Layout section updated to describe the three
  folders by their actual current state (`shared/` ships with a
  README, `packages/` ships with a README, `libraries/` doesn't
  ship at all and materialises on demand).
* `CONTRIBUTING.md` — "libs/ vs libraries/" section retitled to
  "shared/ vs libraries/", table updated, resolution-order prose
  flipped.  Tool-owned-files paragraph now lists `shared/` instead
  of `libs/`.
* `AGENTS.md` — File-ownership table replaces the `libs/` row
  with three rows for `shared/`, `packages/`, `libraries/` (the
  last marked as lazy-created).
* `workspace.yml` — `library_sources:` block comment updated to
  reflect the new fall-through order (`shared/` + per-library
  `src/` + `packages/`).
* `pyproject.toml` — `[tool.ruff].extend-exclude` swaps `libs` for
  `shared`.

Tested
======

3 template tests pass against chumicro-workspace 0.2.0 (mono-repo
`283918b` provides the renamed `WorkspaceLayout.shared_dir` +
search-path adjustments).

After this commit
=================

A user on the previous template needs to manually:

```bash
git mv libs shared
```

if they care about preserving any user code under `libs/`.  Otherwise
a fresh clone or `python run.py update` (once the upstream-template
update flow tracks this rename) is the migration story.  No
automated migration helper since the repo is private and pre-publish.
