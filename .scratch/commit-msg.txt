_templates/ -> _workspace_template/, surface dump-config in the README quickstart

Two audit-of-the-audit follow-ups in one commit (template-repo half).
Mono-repo half landed in `6feb22d`.

A. dump-config surfaced in quickstart + worked example
======================================================

The `python run.py dump-config <project>` command always existed
and is documented in `CONTRIBUTING.md` after the config-flow
diagram, but the `README.md` quickstart never mentioned it — so a
beginner whose `!secret` resolves wrong or whose key landed in the
wrong section had no obvious "introspect the merged result" path.

* `README.md` Quickstart — adds a `dump-config` line between the
  edit-step and the deploy-step ("optional preview the merged
  config the device will read").
* `README.md` worked-example walkthrough — adds a step 5 calling
  `dump-config` before the actual deploy in step 6.

The new `MissingConfigKey` message in chumicro-config (mono-repo
`6feb22d`) hints at the same command when a required key is missing
on device, so the user catches the same problem from either side.

D. _templates/ -> _workspace_template/
======================================

`_templates/` (tool-owned fixture sources rewritten by `update`)
sat next to `projects/_template/` (the user-copied scaffold) at the
workspace root.  Singular vs plural, both leading-underscore-hidden,
both visible in `ls`.  The audit named it as a real collision
point — the tool-owned one gets the more-explicit name.

* `git mv _templates _workspace_template` — directory rename
  carries `devices.yml` + `secrets.yml` to the new location.
* `README.md`, `AGENTS.md`, `CONTRIBUTING.md` — every prose
  reference flipped from `_templates/` to `_workspace_template/`.
* `_workspace_template/secrets.yml` (the materialized template's
  own header comment) — flips the self-reference.

After this commit
=================

A user on the previous template needs to manually:

```bash
git mv _templates _workspace_template
```

if they care about preserving any local edits to the template
sources.  Otherwise a fresh clone or `python run.py update` (once
the upstream-template update flow tracks this rename) is the
migration story.  No automated migration helper since the repo
is private and pre-publish — same precedent as
Decision 0046's `libs/` -> `shared/` rename.

Tested
======

3 template tests pass against chumicro-workspace 0.3.0 (mono-repo
`6feb22d` ships the renamed `_workspace_template/` zone classifier
+ `materialize_templates` directory pointer + `template_apply` /
`health.py` / `cli.py` references).
