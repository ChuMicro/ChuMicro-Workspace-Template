Two-file config (chumicro Decision 0057): one gitignored workspace.yml + per-project config.toml

Tracks the chumicro-workspace 0.9.0 release, which collapsed the
prior 4-layer config overlay (committed workspace.yml + gitignored
workspace.local.yml + committed config.toml + optional gitignored
config.local.<suffix>) to two gitignored layers: workspace.yml
holds workspace-wide defaults *plus* credentials in one place; the
per-project config.toml is gitignored when scaffolded by `new`.
Both deep-merge with the same primitive every other config layer
uses; no `!secret` marker, no resolver, no overlay file.

Workspace-template changes:

- pyproject.toml — chumicro-workspace floor 0.8.0 → 0.9.0.
- workspace.yml — `git rm` from the template; setup materializes it
  from the chumicro-workspace package's canonical starter on first
  run, so each clone gets its own gitignored copy (carrying that
  user's wifi password / broker auth).
- .gitignore — drop the now-redundant `workspace.local.yml`,
  `projects/**/config.local.<suffix>`, and legacy `secrets.yml`
  patterns; add `workspace.yml`.
- README, AGENTS, CONTRIBUTING — rewrite the credential-wiring
  walkthrough for the 1-file shape; trim the merge-pipeline diagram
  from 4 layers to 2; drop `workspace.local.yml` from file-ownership
  tables and "don't commit" lists.
- .github/skills/add-new-project/SKILL.md — same; the "wire
  credentials" step now points at the gitignored `workspace.yml`
  directly.
- projects/_template/config.toml + projects/example_sensor/config.toml
  + every examples/*/config.toml — header comments rewritten for the
  2-layer shape.
- projects/example_sensor/app.py — module docstring trimmed.
- examples/wifi_only/{app.py,README.md} + examples/hello_world/README.md
  + examples/two_projects/README.md — same vocabulary swap.

Migration for existing workspaces (sole-user at the time of this
change; no compat shim):

1. Copy any `defaults:` from `workspace.local.yml` into `workspace.yml`.
2. Copy any `defaults:` from `config.local.<suffix>` into the matching
   per-project `config.toml` (or up into `workspace.yml`).
3. `git rm --cached workspace.yml` if your clone had it tracked from
   a pre-0.9.0 era.
4. Delete `workspace.local.yml` and any `config.local.<suffix>`.
5. Run `python3 run.py setup` to refresh the venv against
   chumicro-workspace 0.9.0.

Tests pass: workspace-level smoke (`tests/test_workspace.py`).
